## Goal
An ns16550a-compatible UART at `0x1000_0000` complete enough that the Linux 8250 driver
(`CONFIG_SERIAL_8250`) drives it as `ttyS0` for console, getty, and full-screen curses apps
— including the THR-empty interrupt dance that most half-baked emulated UARTs get wrong.

## Context
Byte-wide registers, `reg-shift = 0`, offsets 0–7: RBR/THR/DLL(0), IER/DLM(1), IIR/FCR(2),
LCR(3), MCR(4), LSR(5), MSR(6), SCR(7); DLAB (LCR bit 7) banks in the divisor latch. LSR
bits: DR=0x01, OE=0x02, THRE=0x20, TEMT=0x40. IER bits: ERBFI=0x01 (RX), ETBEI=0x02 (THRE).
IIR encoding (priority order): 0x06 line status, 0x04 RX data available, 0x0C character
timeout, 0x02 THR empty, 0x01 none pending; FIFOs-enabled bits 0xC0 when FCR bit 0 set.
The THRE interrupt is edge-ish: it asserts when THR transitions to empty *while ETBEI set*
(and when ETBEI is newly enabled with THR already empty), and *reading IIR when THRE is the
highest-priority source clears it*. RX side: 16-byte FIFO, FCR trigger levels (1/4/8/14),
character-timeout interrupt (0x0C) when FIFO is non-empty below trigger and no activity for
4 char times — Linux relies on this to deliver short lines; without it, input appears only
after 8+ keystrokes. Since emulated output is infinitely fast, THR can drain instantly, but
model TEMT/THRE distinctly. Wire a level-triggered line to PLIC IRQ 10 (E2-T01). Host side
uses the Epic 0 console trait (stdin/stdout native; xterm.js later via E2-T22).

## Deliverables
- `crates/core/src/devices/uart16550.rs` with an explicit interrupt-cause state machine.
- Unit tests scripted at register level: DLAB banking, IIR priority/clearing, FIFO
  trigger/timeout behavior, OE (overrun) when host input floods a full FIFO.
- Bare-metal echo test binary driven interrupt-only (no polling).

## Acceptance criteria
- [ ] Bare-metal interrupt-driven echo works via PLIC claim/complete with IER = 0x03.
- [ ] Register unit tests encode the IIR clearing rules above and pass.
- [ ] Input shorter than the FCR trigger level is still delivered (character timeout).
- [ ] Overrun sets LSR.OE and drops per 16550 semantics rather than corrupting the FIFO.
- [ ] Identical test results native and `wasm32`.

## Adversarial verification
Differential attack: run the same register-poke script (a bare-metal program doing ~50
read/write/IIR-observation steps, including ETBEI toggling with THR empty) on our UART and
on QEMU virt; any IIR/LSR sequence divergence Linux could observe refutes. Flood attack:
inject 10,000 input bytes in one host callback — FIFO must cap at 16 with OE set, interrupt
line must not wedge high after the guest drains. Paste attack (post-E2-T15): paste 100 KB
into a shell — dropped or reordered bytes refute. Starvation attack: mask IRQ 10 at the
PLIC, type, unmask — pending level interrupt must fire exactly once. `stty -F /dev/ttyS0`
under Linux must not error.
