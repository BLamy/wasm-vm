## Goal
A fast, NATIVE, unit-testable answer to "will this bundle actually run?" — so a broken bundle fails
in milliseconds instead of after a ~7-minute guest boot. This advances "run any riscv container" on
the run-side with real (Rust, CI-gated) code, not blind unbootable shell: it validates the exact
bundle `oci unpack` emits and that `wvrun` consumes.

## Context
`wasm-vm oci validate <bundle> [--arch riscv64]` checks: `rootfs/` exists; `run.json` parses; argv[0]
resolves to a REAL regular executable in the rootfs (absolute / via the image PATH / through
symlinks CLAMPED inside the rootfs — a dangling/looping/escaping symlink or a directory is NOT
resolved); and that exec target is arch-correct — an ELF's `e_machine` (parsed directly, no `file(1)`
dependency) matches the target, or a `#!`-script's interpreter (including the `env <prog>` argument)
resolves to a matching ELF. It reuses the T04b matrix critic's lessons in Rust: no false-pass on
dangling/escape, no host-escape via symlinks, wrong-arch (incl. RV32-vs-RV64) rejected.

## Deliverables
- `crates/cli/src/oci/validate.rs` — `validate_bundle` + the `oci validate` subcommand; typed
  `ValidateError`. ELF `e_machine`/class parsed natively.
- Unit tests (13) over synthetic bundles with in-test ELF headers, incl. every adversarial case.

## Acceptance criteria
- [x] A riscv64 bundle validates OK (busybox real bundle → "runnable: sh → 64-bit riscv ELF").
- [x] An amd64 bundle validated `--arch riscv64` is REJECTED (`WrongArch`); the same bundle validates
      OK as `--arch amd64` (real alpine bundle, end-to-end).
- [x] Dangling / rootfs-escaping / looping symlink entrypoints are rejected (not false-passed).
- [x] Script entrypoints resolve their interpreter (incl. `#!/usr/bin/env prog`) and arch-check it.
- [x] Missing rootfs / bad run.json / empty argv / non-ELF-non-script data / unknown arch are typed
      errors, never a panic.

## Adversarial verification
The resolver must not false-pass (the T04b failure mode): a dangling symlink, a directory, a symlink
loop, and a rootfs-escaping relative symlink must all FAIL, and the ELF check must reject a wrong
`e_machine` AND a 32-bit class for a 64-bit target. The `#!/usr/bin/env` path must follow the program
argument, not `file` `env` itself. No path may `open`/`file` a HOST file (symlink clamp). Confirm a
0-byte or truncated ELF header doesn't panic.
