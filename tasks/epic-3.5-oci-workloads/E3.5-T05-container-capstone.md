## Goal
The full loop in a clean browser: boot Alpine, pull a small riscv64 OCI image from a real
registry, run it with the tiny runner and see its output in the terminal; reload the tab,
run it again — this time from the persisted layer cache with zero blob downloads. Then, as
the graduation exercise (gated on guest port-forwarding), a Node/HTTP container serving a
request that a host-browser fetch can hit.

## Context
This is the "civilization ships containers" moment and the proof that E3.5-T01→T04 compose:
importer (pull+verify+unpack) → persistence (cache, reload survival) → runner (namespaces,
overlay, stdio) → all inside the browser-hosted VM. Playwright drives it exactly like the
E3-T05 double-boot test (fresh profile = clean IndexedDB; request interception counts blob
fetches). Image choice: the smallest honest one that proves execution (busybox or
alpine-based, riscv64) — output must be computed in-container (echo-proof marker
discipline: `CONTAINED_$((6*7))`, never a literal the transcript already carries). The
**graduation leg** — a Node (or busybox httpd) container answering HTTP from the host
browser — needs guest→host port-forwarding, which is E6-T25 (or the E3-T16 proxy path run
in reverse); it is an EXPLICIT second checkbox gated on that dependency landing, not a
silent scope cut: record it as deferred-with-pointer if this capstone verifies first.

## Deliverables
- Demo-page surface: a "run container" affordance (dev-panel button or terminal helper
  script in the guest) wired through the T01/T04 importer + T03 runner.
- The Playwright capstone test: fresh profile → boot (chunked, persisted) → pull → run →
  assert computed output → reload → re-run → assert 0 blob fetches + same output.
- Timing/size numbers recorded: first-pull bytes + wall time, cached-run wall time, cache
  size after.
- (Graduation, dep-gated) the HTTP container recipe + test, or the recorded deferral
  pointing at the port-forwarding task.

## Acceptance criteria
- [ ] Fresh browser profile: boot Alpine (lazy chunked + persistent overlay), pull the
      image from a real registry (or the documented CORS proxy), `wvrun` prints the
      computed marker in the terminal.
- [ ] `page.reload()` → boot → same run works with ZERO layer-blob network fetches
      (interception-proven) — image runs fully from IndexedDB/OPFS cache.
- [ ] Total added download for the container feature itself (importer/runner code) is
      accounted and sane; numbers in the log.
- [ ] Kill the tab mid-first-pull; on reopen the pull completes correctly (partial cache
      state either resumes by digest or refetches — never corrupts).
- [ ] (Gated) Node/busybox-httpd container serves HTTP to a host fetch, or the deferral is
      recorded against the port-forwarding task id.

## Adversarial verification
Run the capstone in a genuinely fresh profile (no warm state) — a stale-profile pass
refutes nothing. Interception must count ALL network bytes, not just XHR (fetch + workers).
Prove the second run is the cache and not the registry: run it offline (context.setOffline)
— it must still work. Tamper one cached layer before the second run — it must refetch (or
fail typed offline), never run corrupted code. Diff the in-container marker computation
against a literal-echo trap (the E3-T13 F1 lesson: transcript echo makes `contains()`
vacuous — markers must be computed in-guest).
