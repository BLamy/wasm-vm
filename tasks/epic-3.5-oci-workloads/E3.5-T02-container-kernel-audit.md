## Goal
An audited, documented kernel configuration for running containers in the guest: confirm
(and where missing, enable + rebuild) namespaces, cgroups, seccomp, overlayfs, veth/bridge/
NAT, tmpfs, and loop devices — with an in-guest proof for each (not just a config grep).

## Context
**Audited 2026-07-06 against the shipped `releases/kernel/6.6.63/config`** (the E3-T13
network-enabled rebuild pulled most of this in via the riscv defconfig):
ALREADY ON — `NAMESPACES` + `PID/NET/USER/IPC/UTS_NS`, `CGROUPS` + `MEMCG/CPUSETS/
CGROUP_PIDS/CGROUP_CPUACCT`, `SECCOMP` + `SECCOMP_FILTER`, `OVERLAY_FS`, `VETH`, `BRIDGE`,
`NETFILTER` + `NF_NAT`/`IP_NF_NAT`, `TMPFS`, `BLK_DEV_LOOP`.
CURRENTLY OFF — `TUN` (needed for some container network modes; not for veth/bridge v1)
and `SQUASHFS` (only if we ship squashed layers). Decide each explicitly: enable in
`configs/wasm-vm.config` + rebuild via `tools/build-kernel.sh`, or document why deferred.
Also verify the runtime side: cgroup v2 mount works (`/sys/fs/cgroup`), `unshare`/`nsenter`
exist in the guest userland (busybox has unshare; Alpine's util-linux if not), and
`pivot_root` into an overlayfs-backed rootfs works on our emulated MMU — an emulator bug
in any syscall here (clone3 flags, setns, pivot_root, mount) surfaces NOW rather than
mid-runner (E3.5-T03).

## Deliverables
- `docs/containers.md`: the audited config matrix (symbol → state → why), the TUN/SQUASHFS
  decisions, and the in-guest proof transcript for each capability.
- Any `configs/wasm-vm.config` additions + rebuilt pinned kernel (if a symbol is enabled).
- An in-guest smoke script (`tools/guest/container-smoke.sh`, shipped into the rootfs):
  unshare each namespace type, write a cgroup limit and see it enforced, load a trivial
  seccomp filter, mount overlayfs (lower+upper+work), create a veth pair into a bridge,
  mount tmpfs, loop-mount an image file — each step prints PASS/FAIL.
- Native expect-test running the smoke script in a booted Alpine and asserting all-PASS.

## Acceptance criteria
- [ ] The smoke script passes every step in a booted guest (native, transcript in the log).
- [ ] `unshare -p -m -n -u -i -f --mount-proc sh -c 'echo NS_OK'` works (the T03 runner's
      exact shape).
- [ ] overlayfs mount over tmpfs upper + ext4 lower serves reads/writes/whiteouts correctly
      (create/delete/modify across layers).
- [ ] A memory cgroup limit demonstrably kills an over-allocating process (OOM in the
      group, guest stays alive).
- [ ] TUN + SQUASHFS each has a recorded decision (enabled + rebuilt, or deferred with the
      task that needs it named).

## Adversarial verification
The grep is not the proof — every "=y" claim must be exercised in-guest (an emulator
missing a syscall makes the config a lie). Specifically hunt emulator gaps: clone3 with
namespace flag combinations, setns across all types, pivot_root + umount2(DETACH) inside a
mount namespace, seccomp SECCOMP_SET_MODE_FILTER actually filtering (a blocked syscall
returns EPERM/kills, not silently succeeds). Any step that fails IN THE GUEST but works on
real riscv64 Linux is an emulator bug to file + fix, and refutes until triaged.
