## Goal
Pulled OCI layers persist in the browser: a content-addressed layer cache in IndexedDB (or
the OPFS backend when E3-T06 lands) keyed by digest, deduped across images, surviving a tab
reload — so the second `pull` of anything already cached transfers zero bytes.

## Context
This is the E3-T05 machinery pointed at a second store: blobs are immutable and
content-addressed (sha256 digest = key), which is the EASY case — no generations, no
write-back races, just put-once/get-forever plus verified-on-read (re-hash on load; a
corrupted stored blob is evicted + refetched, never trusted). Schema: a DB (or store)
separate from the disk overlay — `oci-blobs` (key = digest, value = compressed blob bytes)
+ `oci-meta` (manifests/indexes by digest, tag → digest resolution records with a
staleness note). Storage pressure is real (base images are tens of MB compressed):
integrate with the E3-T10 quota story — surface cache size, LRU-evict UNPINNED blobs on
pressure (a pinned image = one the user ran), `navigator.storage.persist()` already
requested by the overlay path. Dedupe means: pulling `alpine:3.20` after `alpine:latest`
fetches only the layers whose digests differ (usually zero). The unpacked-rootfs
materialization (cache → guest-visible tree) reuses T01's applier; whether unpacked trees
are also cached (space/time trade) is a documented decision, not an accident.

## Deliverables
- Browser layer store (IndexedDB via the E3-T05 `IdbStore` pattern; store names versioned)
  with put/get/has/delete by digest + verified-on-read.
- Pull-through wiring: T01's browser importer checks the cache before any network fetch;
  hits are logged + counted (`cacheStats` JS surface like E3-T02's `fetchStats`).
- Dedupe proof + eviction: LRU over unpinned blobs with a byte budget; pin/unpin API.
- Reload survival test (Playwright, the E3-T05 double-boot pattern): pull → reload →
  re-pull is 100% cache hits, zero network blob fetches (assert via request interception).

## Acceptance criteria
- [ ] Pull a small riscv64 image in-browser → reload the tab → pull again: 0 blob bytes
      over the network (Playwright intercepts prove it), manifest re-resolution allowed.
- [ ] Two images sharing base layers store each shared layer ONCE (store size accounting
      proves dedupe).
- [ ] A stored blob corrupted via DevTools/IDB tampering is detected on read (digest
      re-check), evicted, and refetched — never unpacked.
- [ ] Eviction under a tiny byte budget removes only unpinned LRU blobs; pinned images
      remain runnable offline.
- [ ] Cache size + hit/miss counters surfaced to JS (and visible in the demo page's boot
      panel).

## Adversarial verification
Tamper the IDB store directly (wrong bytes under a correct key, right bytes under a wrong
key, truncated value) — any path that unpacks unverified bytes refutes. Fill to quota
mid-pull: the typed StorageFull path must surface (E3-T10 interplay), no partial-layer
state that later reads as valid. Concurrent pulls of the same digest (two tabs or two
in-flight fetches) must not corrupt the store (last-write-wins on identical bytes is fine).
Kill the tab mid-put; on reopen the digest either exists-and-verifies or is absent.
