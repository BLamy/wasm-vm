## Goal
NOT Docker Engine. A small in-guest runner (`wvrun`) that takes an unpacked OCI rootfs +
image config and runs the container: set env/cwd/entrypoint+cmd from the config, mount
proc/sys/dev(+pts) inside, isolate with the namespaces/cgroups E3.5-T02 proved, and wire
the container's stdio straight to the terminal (interactive `sh` in a container must feel
like a shell).

## Context
Scope discipline is the task: runc exists and is huge; we need the ~20% that runs real
images. Start shape: a static riscv64 binary (Rust, musl static, built by our Docker
toolchain — or a POSIX sh script over `unshare`/`mount` for v0 if that proves faster to
verify) living in the guest rootfs. Sequence: read `config.json`-equivalent (we generate a
simplified runtime config from the OCI image config at unpack time — T01 owns that
translation), `unshare(PID|NS|UTS|IPC|MNT[,USER])`, mount overlayfs (image layers as lower,
tmpfs or overlay-disk upper — container writes must not mutate the image), bind /dev
essentials + fresh proc/sys, `pivot_root`, apply cwd/env, drop to entrypoint via `execve`.
Cgroups: create a leaf group per container, apply memory/pids limits when the config asks.
Rootless (USER_NS + uid_map) is the stretch — root-in-guest is acceptable v1 (the guest IS
the sandbox). Networking: v1 containers share the guest netns (loopback works, eth0 visible)
— veth-per-container arrives with T05's graduation, not here. No image pulling here (T01),
no daemon, no exec/attach API — run-to-exit + interactive tty only.

## Deliverables
- `wvrun` in the guest image (source under `guest/wvrun/` or `tools/guest/`, built by the
  rootfs pipeline): `wvrun <bundle-dir>` and `wvrun --interactive <bundle-dir>`.
- T01's unpack emits the bundle: `rootfs/` + the simplified runtime config (entrypoint,
  cmd, env, cwd, user).
- Exit-code fidelity: the container's exit code IS wvrun's exit code (incl. 137-style
  signal deaths).
- Native expect-test: boot Alpine, `wvrun` a busybox-image bundle, assert entrypoint output,
  env/cwd correctness, exit-code passthrough, and that container writes did NOT reach the
  image layers (overlay upper only).

## Acceptance criteria
- [ ] `wvrun` runs the unpacked alpine image's `/bin/sh -c 'echo CONTAINED_$((6*7))'` with
      the image's env/cwd → prints `CONTAINED_42`, exit 0 (echo-proof marker discipline).
- [ ] Inside the container: `ps` shows PID 1 = the entrypoint (PID ns + fresh proc), the
      hostname differs (UTS ns), and the image rootfs is `/` (pivot_root) with proc/sys/dev
      mounted.
- [ ] Container writes to `/` land in the overlay upper; the unpacked image layers are
      byte-unchanged after the run (hash the layer tree before/after).
- [ ] A memory-limited container that over-allocates is OOM-killed; the guest and a
      subsequent `wvrun` are unaffected.
- [ ] Interactive: `wvrun --interactive` gives a usable shell over the VM terminal (native
      expect proof: type, see output, exit propagates).
- [ ] **Seccomp (relocated from E3.5-T02):** the runner installs a runc-style seccomp filter
      (`SECCOMP_SET_MODE_FILTER`) before exec, and a syscall the profile denies demonstrably
      returns EPERM / kills the process inside the container — proving `SECCOMP_FILTER=y` is
      real on the emulator, not merely configured. A blocked syscall that silently succeeds
      is an emulator gap to file + fix.

## Adversarial verification
Escape-hunt: from inside the container, attempt to read a guest-only file (`/root/secret`
planted outside the bundle), remount /, umount the overlay, `kill -9` guest PIDs, exhaust
PIDs against a pids-limit — any success refutes the corresponding isolation claim (document
which isolations are NOT claimed in v1, e.g. no user-ns → root inside == root in guest; the
charter is honesty, not perfect confinement). Kill wvrun mid-run: no leaked mounts/cgroups
after (verify `/proc/mounts`, cgroup dir gone). Malformed bundle (missing entrypoint,
non-exec entrypoint, config/rootfs mismatch) → typed error, never a guest panic.
