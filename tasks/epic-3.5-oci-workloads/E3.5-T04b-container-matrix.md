## Goal
Brett's directive: be **generic enough to run any Docker container that is riscv-compatible**, and
PROVE it — not just claim it. Turn "the pipeline worked on busybox + postgres" into "the pipeline is
verified across a spread of real riscv64 images (minimal userland, web servers, datastores,
proxies)."

## Context
`sideload → oci unpack → bundle → wvrun` is already image-agnostic (nothing image-specific in the
code). The proof splits in two, and this env can only run one half:
- **Natively verifiable HERE (no boot):** pull + digest-verify + whiteout-unpack + a coherent,
  correct-arch runnable bundle. `tools/oci-matrix.sh` runs the real path per image and statically
  asserts: unpack OK; `run.json` argv[0] resolves in the rootfs (absolute / via the image PATH /
  through symlinks like `/bin/sh → busybox`); if argv[0] is an ELF it is RISC-V, if a #!-script its
  interpreter resolves and is RISC-V; and ARCH PURITY — every ELF in the standard bin dirs is
  RISC-V, no foreign-arch binary (the real "did we pull the right arch and is the image actually
  riscv-native" check).
- **Boot-gated (DEFERRED — this env kills long interpreter boots):** actually `wvrun` each image and
  observe it run/serve. Needs the "bundle into the guest" mechanism (a second virtio-blk disk
  carrying the bundle rootfs as ext4, mounted in-guest) — shared with the E3.5-T05 capstone.

riscv64 availability survey (Docker Hub, 2026-07-06): **HAVE riscv64** — alpine, busybox, nginx,
httpd, caddy, haproxy, redis, memcached, postgres, traefik, debian, ubuntu, python, ruby, rust,
golang. **NO riscv64 (yet)** — node, mariadb, mysql, mongo. So "any riscv-compatible container" is a
real, large set; the tool must fail cleanly on the non-riscv ones (it does: "no <arch> manifest").

## Deliverables
- `tools/oci-matrix.sh [image ...]` — per-image pull→unpack→coherence+arch-purity, a markdown report
  table, non-zero exit on any failure. Curated default spread.
- A committed report snapshot (`docs/container-matrix.md`) from a real run.
- Deferred boot-matrix acceptance (`#[ignore]`): boot once, `wvrun` each bundled image, assert its
  own in-container marker (echo-proof) — gated on the bundle-into-guest mechanism (T05).
- Any pipeline generality fixes the matrix surfaces (e.g. zstd layers, extra device nodes) filed +
  fixed or ticketed.

## Acceptance criteria
- [x] `tools/oci-matrix.sh` PASSES the curated riscv64 spread natively — **9/9** (alpine, busybox,
      nginx, httpd, caddy, haproxy, redis, memcached, postgres); report in `docs/container-matrix.md`.
- [x] A non-riscv image fails cleanly: node/mariadb/mysql/mongo → "no riscv64 manifest in the index",
      non-zero exit, no crash (survey + resolver path).
- [x] Arch-purity check proven NON-vacuous: an amd64 alpine's busybox reads `x86-64` → FAIL (so a
      riscv64 PASS is meaningful).
- [ ] Any image whose layers use zstd unpacks — none in this spread hit zstd (all gzip layers);
      **ticketed as a generality gap** (unpack is gzip-only) for a follow-up.
- [ ] (Deferred/boot) at least one non-trivial server image (nginx or redis) `wvrun`s and answers
      in-guest — recorded transcript when a long-boot env is available.

## Adversarial verification
The arch-purity check must actually catch a wrong-arch image — verify by pointing the resolver at an
amd64 bundle (or a hand-planted x86 ELF) and confirming FAIL, so PASS is not vacuous. The
argv[0] resolver must not false-PASS on an unresolved entrypoint (symlink chase must terminate; a
missing binary must FAIL, not silently pass). Confirm the tool fails, not hangs, on an image with no
riscv64 manifest and on a registry error.

## Verification log

**2026-07-06 — matrix PASSES 9/9 riscv64 images natively (pass 1).** `tools/oci-matrix.sh` ran the
real pipeline (`oci-sideload` pull+digest-verify → `wasm-vm oci unpack` → bundle) over a spread of
runtime types and statically proved each is a coherent, correct-arch runnable container WITHOUT a
boot: **alpine, busybox** (userland), **nginx, httpd, caddy** (web servers), **haproxy** (proxy),
**redis, memcached, postgres** (datastores). Every image: entrypoint resolves in the rootfs (abs /
via image PATH / through symlinks like `/bin/sh → busybox` — a resolver bug where absolute symlinks
resolved against the HOST was fixed with an in-rootfs symlink `chase`), and ARCH PURITY holds — 7 to
400 ELFs per image in the bin dirs, ALL riscv64, zero foreign. Report: `docs/container-matrix.md`.
**Anti-vacuous proof:** pulled alpine for **amd64**, its busybox reads `x86-64` and the purity regex
flags FOREIGN → the matrix would FAIL it, so a riscv64 PASS is real. **Fails cleanly on non-riscv:**
node/mariadb/mysql/mongo have no riscv64 manifest → "no riscv64 manifest in the index", non-zero
exit. **GENERALITY GAP surfaced (ticketed):** `oci unpack` decompresses gzip only (sniffs `1f 8b`);
a zstd-compressed layer would be mis-read as a raw tar and fail — none in this spread used zstd, but
some modern images do → follow-up. Boot leg (`wvrun` each) deferred (needs bundle-into-guest + long
boot). Cold-clone critic on this diff: next.
