## Goal
Toward "run any riscv-compatible container": the sideload pulled from Docker Hub only (hardcoded
`registry-1.docker.io` + `auth.docker.io`). Many images live on **ghcr.io / quay.io / gcr.io /
public.ecr.aws / private registries**. Support any standard Docker v2 registry with anonymous pull.

## Context
Instead of hardcoding each registry's endpoints, use the **standard v2 auth challenge**: make the
request; on a 401, read `WWW-Authenticate: Bearer realm=…,service=…,scope=…` from THAT response and
fetch a token from its realm. This auto-works for every compliant registry. Two real bugs surfaced
and were fixed during development:
1. **Silent death on a no-auth registry.** The first cut pre-probed `/v2/…/manifests/tag` with HEAD
   and `grep`ed for the challenge; a public registry (quay's HEAD) answers 200 with NO challenge →
   `grep` exits 1 → under `set -e` the whole script died with zero output. Docker Hub masked it (it
   always 401s). Fixed: challenge ON the actual GET (quay allows an unauth HEAD but 401s the GET, so
   the pre-probe was wrong anyway), and the token fetch tolerates a no-challenge 200.
2. **`nginx:1.27` mis-parsed as a host.** Ref parsing treated the first `/`-segment as a host when it
   had a `:`; with no slash, the whole `nginx:1.27` became "host `nginx:1.27`". Fixed: a host only
   exists when the ref contains a `/`. **Caught by the deterministic `--selftest`, not the live
   tests** (which all used slash-bearing or colon-free refs).

## Deliverables
- `oci-sideload.sh` parses `[host/]repo[:tag]` (Docker's rules) + challenge-based auth for any v2
  registry; ghcr `{token}` / others' `{access_token}` both accepted.
- `parse_ref` extracted as a pure function + `oci-sideload.sh --selftest` (7 deterministic cases).
- `crates/cli/tests/oci_sideload_parse.rs` runs `--selftest` so CI gates the parser (no network).

## Acceptance criteria
- [x] Docker Hub still works end-to-end (busybox riscv64 → 4 blobs → unpack) — regression.
- [x] A second registry works end-to-end: `quay.io/podman/hello` → token challenge on the GET →
      4 blobs → `oci unpack` OK (amd64; proves the registry+auth path independent of arch).
- [x] ghcr.io token challenge succeeds (auth flow reached; a wrong image name returns a clean 404,
      not an auth error).
- [x] `--selftest` passes 7 ref-parse cases incl. bare `nginx:1.27`, `ghcr.io/owner/img`,
      `gcr.io/proj/sub/img:tag`, `localhost:5000/img:v2`; gated in CI via the Rust test.
- [ ] A riscv64 image on a non-hub registry unpacks end-to-end — none readily found (most non-hub
      images are amd64/arm); the registry path is proven on quay (amd64) + arch on hub (riscv64).

## Adversarial verification
The parser must not mis-split `host:port`, a `:tag`, or a `/`-bearing repo (all in `--selftest`).
The auth path must: fetch the token from the challenge's OWN realm (not a hardcoded one), survive a
mid-pull 401 (re-challenge), tolerate a no-auth 200, and STILL digest-verify every blob (the
untrusted-registry guarantee is unchanged — a non-hub registry is no more trusted than Docker Hub).
Confirm a bad token / wrong scope fails loudly (non-2xx logged), not silently.
