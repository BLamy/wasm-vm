# E3-T10 browser evidence

Generated by the opt-in Playwright acceptance tests against a freshly built `web/` tree.

- `quota-dialog-after-retry.png` — real IndexedDB transaction abort classified as
  `QuotaExceededError`; Retry reaches the same storage-full dialog without losing the pending
  batch.
- `pristine-after-reset.png` — third boot after typed reset, with the pristine marker and zero
  matching ext4 error/warning/corruption lines.
- `browser-summary.json` — machine-readable quota, IOERR exit code, recovery, reset, and console
  assertions from the three-boot proof.
- `browser-console-errors.txt` — expected to be empty.
- `browser-demo-126-of-126.png` — full-page demo proof after all 126 in-browser RISC-V binaries
  pass, with the E3-T10 roadmap evidence visible.

Commands:

```sh
make web-build
cd web
E3_T10_FULL=1 npx playwright test tests/quota.spec.js --grep "forced quota"
E3_T10_DEMO=1 npx playwright test tests/e3-t10-demo-proof.spec.js
```
