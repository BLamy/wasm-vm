# E3-T10 browser evidence

Generated by the opt-in Playwright acceptance tests against a freshly built `web/` tree.

- `quota-dialog-after-retry.png` — real IndexedDB transaction abort classified as
  `QuotaExceededError`; Retry reaches the same storage-full dialog while the write remains
  unacknowledged.
- `quota-continue-ioerr.png` — terminal proof that Continue completes the parked request with EIO,
  makes `dd` fail, and leaves the guest usable.
- `quota-reopen-fsck.png` — exact completed-byte/hash comparison after reopen.
- `quota-fsck.txt` — offline journal replay plus `e2fsck -f -n` over the exact reconstructed
  base-image + persisted IndexedDB overlay snapshot captured after the quota-edge tab close.
- `pristine-after-reset.png` — third-boot terminal after typed reset, with the pristine marker and
  zero matching ext4 error/warning/corruption lines.
- `quota-terminal.txt` — full guest console transcript across all three boots, prefixed with the
  source commit and Playwright-trace SHA-256.
- `quota-playwright-trace.zip` — reopenable Playwright snapshots/screenshots/sources for the whole
  quota, tab-close, reopen, fsck, idle, reset, and pristine sequence.
- `browser-summary.json` — machine-readable source commit, quota/IOERR result, dd-completed bytes,
  reopened content hash, fsck, idle-zero-backlog, reset, trace digest, and console assertions.
- `browser-console-errors.txt` — expected to be empty.
- `browser-demo-126-of-126.png` — full-page demo proof after all 126 in-browser RISC-V binaries
  pass, with the E3-T10 roadmap evidence visible.

Commands:

```sh
make web-build
cd web
E3_T10_FULL=1 npx playwright test tests/quota.spec.js --grep "forced quota" --trace off
E3_T10_DEMO=1 npx playwright test tests/e3-t10-demo-proof.spec.js
```
