# Container matrix — the OCI pipeline runs any riscv-compatible image (E3.5-T04b)

Evidence that `sideload → oci unpack → bundle` is **generic**, not postgres-specific. Each
image is pulled (digest-verified), unpacked to a runnable bundle, and statically checked:
the entrypoint resolves in the rootfs (following the image PATH + symlinks), and **arch
purity** — every ELF in the standard bin dirs is RISC-V, no foreign binary. Regenerate with
`tools/oci-matrix.sh`. The check is not vacuous: an amd64 image FAILs (its busybox reads
`x86-64`). The `wvrun` **boot** of each image is the deferred half (needs the bundle-into-
guest disk + a long interpreter boot this CI env kills).

## riscv64 availability (Docker Hub survey, 2026-07-06)

**Publish riscv64:** alpine, busybox, nginx, httpd, caddy, haproxy, redis, memcached,
postgres, traefik, debian, ubuntu, python, ruby, rust, golang.
**No riscv64 yet:** node, mariadb, mysql, mongo — `oci-sideload` fails cleanly on these
("no riscv64 manifest in the index", non-zero exit), never a crash.
